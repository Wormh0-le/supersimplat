# Stage Generated View quality gates before publication

The first Companion renderer will process Generated Views incrementally. After camera preflight, each candidate joins a temporary SAM3 prefix replay and must pass a hard structural gate before a versioned neighbor-anomaly gate compares its mask area, bounding-box center, projected Seed Region overlap, contributor support, and SAM3 object confidence with accepted neighboring views. Rejected views remain neutral and any replacement follows the same path; an Anchor failure fails the whole update. Thresholds are explicit policy-revision data calibrated by controlled fixtures, not implicit heuristics. Temporary frames, tracks, and diagnostics never publish. Once selection is final, the complete selected Frame Set is replayed from the authoritative Prompt Log and published atomically. Three consecutive accepted views with less than two percent incremental contributor coverage stop all later rendering attempts.

The first executable revision is `generated-view-neighbor-anomaly/v1`. It compares each hidden candidate with the immediately preceding accepted view, beginning with the Anchor. Projected Seed Region overlap has both an absolute floor and a relative minimum/maximum against that neighbor; because the Seed Region is derived from the accepted Anchor mask, the Anchor starts with a defined overlap baseline of `1.0`. Missing projected Seed Region or adapter confidence metrics are neutral rejections rather than a bypass. Its threshold values, measured inputs, and reasons remain internal/benchmark artifacts in [`generated-view-neighbor-anomaly-v1.json`](../benchmarks/fixtures/generated-view-neighbor-anomaly-v1.json); changing calibration requires a new policy revision and matching fixture evidence.
