# Controlled front/back-overlap fixture

Deterministic 16,384-Gaussian fixture for the controlled-overlap PoC benchmark: an 8,192-Gaussian red target ellipsoid shell and an 8,192-Gaussian blue distractor shell offset behind it (+z), so the target's rear is observable only by orbiting through the gap between the shells while the distractor stresses back-layer leakage. Generated by `scripts/benchmarks/generate_controlled_overlap.py` at seed 20260712; the scene PLY is frozen (`sha256 cb238cb7…`).

## Ground Truth revision 2 (ADR 0011)

Rear-surface Ground Truth is the **honestly observable rear surface**, not the mechanical `z > 0` half. A fixed geometric screening rule — `geometric-sight-line-screening/v1`, recorded in the Ground Truth JSON — classifies a target Gaussian as **ambiguous** if and only if no direction in a fixed 1024-direction Fibonacci sphere gives it an unobstructed sight line past every distractor Gaussian (ray-sphere intersection at the distractor set's covering radius 0.022093). The rule reads fixture geometry only; it is independent of any method, trial result, or the Evidence Policy.

- selected truth: 7,904 IDs; rejected truth: 8,192 distractor IDs; ambiguous truth: 288 IDs (the distractor-enclosed rear cap, z ∈ [0.311, 0.344]).
- rear surface: 3,807 IDs (the `z > 0` half minus the ambiguous cap); rear recall is gated at ≥ 0.90 against this observable set.
- **Disclosure:** the 288 ambiguous Gaussians are enclosed by the ≥0.96-opacity distractor shell from every external sight line; no honest method can gather selection-strength evidence for them. Candidates still classify them (they remain part of the scene), but accuracy metrics exclude them per Benchmark Ground Truth semantics. See ADR 0011 for measurements, cross-checks against the sealed multipoint-v3 diagnosis (482-cap), and the rejected alternatives.

Revision 1 (empty ambiguous set, mechanical `z > 0` rear) is superseded; the sealed multipoint-v3 trial record remains valid historical evidence under it. Predictions sealed under revision 1 are not re-scored against revision 2.

## Re-freezing

The Ground Truth is reproducible from the generator. Freeze order matters because the repository holds Prettier-canonical JSON and the manifest records file digests:

```sh
uv run --project selection-service-companion --locked --python 3.12 \
  python scripts/benchmarks/generate_controlled_overlap.py --output-dir <dir>
npx prettier --write <dir>/controlled_front_back_overlap_ground_truth.json
uv run --project selection-service-companion --locked --python 3.12 \
  python scripts/benchmarks/generate_controlled_overlap.py --output-dir <dir> --refresh-manifest-digests
npx prettier --write <dir>/controlled_front_back_overlap.json
```

The generator prints the occlusion radius and the screening cross-check against the measured 482-cap at every run. The PLY output is byte-identical across runs; the npz embeds a zip timestamp, so npz identity is array-level, not byte-level.
